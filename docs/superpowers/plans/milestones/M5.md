## MILESTONE 5 — Costura ai-jail + trecho CLAUDE.md

Objetivo: fechar a **costura** entre o `ai-jail` (confinamento de processo, caixa-preta versionada) e o engine PRÓPRIO do produto. Um único comando (`sbx shell`) sobe o supervisor da sessão (M4), expõe **só** o socket podman do produto dentro de uma `ai-jail --no-docker` e roda o agente já cabeado a esse engine — com o Docker/socket/credenciais do host invisíveis. Um segundo comando (`sbx bootstrap-claude-md`) imprime um bloco de CLAUDE.md pronto para colar, que ensina o agente a usar as primitivas com as regras de segurança. Por fim, um polish de `--help`/exemplos garante que um LLM use a ferramenta corretamente só lendo `--help`.

Esta é a implementação da topologia **"ao lado"** do ADR 0001 §2.1: o engine roda **fora** do `bwrap` (na conta do usuário, com storage/runroot/socket próprios) e o agente sob ai-jail alcança **apenas** esse socket via `--rw-map`.

**Consome de M4 (exatamente como M4 define):**
- `session.OpenRegistry(sessionID) (*Registry, error)` — registry persistente.
- `session.DestroyAll(ctx, drv, sessionID) error` — cleanup idempotente (registry ∪ engine). **Não** reimplementar um helper local `destroyAllInSession` que só faz `List`+`Destroy`.
- Hooks de proxy do CLI (`startSessionProxy` / `stopSessionProxy` de M4.5) — o `sbx shell` embute o papel de supervisor **in-process** (sobe proxy, grava `ProxyAddr`, cleanup com `DestroyAll` + stop proxy + `MarkEnded`); `sbx session start`/`end` continuam para modo detached.

**Pré-requisitos de ambiente:**
- `ai-jail` instalado e no `PATH` (binário externo — decisão D1 do ADR: pinado, não forkado).
- `podman` rootless (mesmos pré-requisitos de M1).
- Kernel Linux (nativo ou WSL2 com `systemd=true`).

**Estratégia de teste:** `aijail.Recipe`/`ValidateSocket`, `Podman.ServiceArgs`, os `baseArgs` remotos e o gerador de CLAUDE.md são **unitários determinísticos** (asserção de flags/conteúdo, sem engine). A montagem do argv de `sbx shell` é testada por uma função pura (`shellPlan`). O ponta-a-ponta sob um `ai-jail` real é **integração/manual**, atrás de `//go:build integration` com `t.Skip` se `ai-jail`/`podman` ausentes.

> **Nota de nomes de arquivo:** a File Structure do plano lista `internal/aijail/compose.go`; como este código não tem relação com compose, os arquivos deste milestone são `internal/aijail/recipe.go` e `internal/aijail/claudemd.go` (o pacote e as funções pinadas — `aijail`, `Recipe`, `ValidateSocket` — permanecem exatos).

// sbx/internal/aijail/recipe.go
package aijail

import (
	"path/filepath"

	"github.com/gustavocarvalho/sbx/internal/driver"
)

// hostDockerSockets são caminhos que NUNCA podem ser expostos ao agente:
// acesso ao socket do Docker do host é equivalente a root no host.
var hostDockerSockets = map[string]bool{
	"/var/run/docker.sock": true,
	"/run/docker.sock":     true,
}

// ValidateSocket garante que o socket que vamos --rw-map para dentro da jail é
// o socket PRÓPRIO do produto — nunca o socket do Docker do host.
func ValidateSocket(socketPath string) error {
	if socketPath == "" {
		return driver.CLIErrorLike("no_socket",
			"no product socket path provided",
			"sbx must expose its own podman socket; refusing to launch the jail without one")
	}
	clean := filepath.Clean(socketPath)
	if filepath.Base(clean) == "docker.sock" || hostDockerSockets[clean] {
		return driver.CLIErrorLike("host_socket_forbidden",
			"refusing to expose a docker socket to the agent",
			"sbx exposes only its own podman socket (…/sbx/<session>/api.sock), never the host Docker socket")
	}
	return nil
}

type config struct {
	masks     []string
	extraArgs []string
}

// Option customiza a receita de invocação do ai-jail.
type Option func(*config)

// WithMask acrescenta padrões extras a esconder do agente (além do default
// ".env*"). O socket do Docker do host é recusado independentemente disso.
func WithMask(patterns ...string) Option {
	return func(c *config) { c.masks = append(c.masks, patterns...) }
}

// WithExtraArgs anexa flags cruas à invocação do ai-jail (ex.: "--lockdown").
func WithExtraArgs(args ...string) Option {
	return func(c *config) { c.extraArgs = append(c.extraArgs, args...) }
}

// Prefix devolve o argv do ai-jail até (e incluindo) o separador "--", pronto
// para o chamador anexar o argv do agente. Devolve nil se o socket for inválido.
func Prefix(socketPath string, opts ...Option) []string {
	if ValidateSocket(socketPath) != nil {
		return nil
	}
	cfg := config{masks: []string{".env*"}}
	for _, o := range opts {
		o(&cfg)
	}
	argv := []string{"ai-jail", "--no-docker", "--rw-map", socketPath}
	for _, m := range cfg.masks {
		argv = append(argv, "--mask", m)
	}
	argv = append(argv, cfg.extraArgs...)
	return append(argv, "--")
}

// Recipe monta o argv para rodar o agente sob ai-jail cabeado ao socket podman
// do produto:
//
//	ai-jail --no-docker --rw-map <socket> --mask .env* [extra] -- sh -c <agentCmd>
//
// Falha fechado: se o socket não for o do produto (ex.: um docker.sock),
// devolve nil. Chamadores DEVEM chamar ValidateSocket antes, para o erro
// acionável.
func Recipe(socketPath, agentCmd string, opts ...Option) []string {
	head := Prefix(socketPath, opts...)
	if head == nil {
		return nil
	}
	if agentCmd == "" {
		agentCmd = "bash"
		return append(head, agentCmd)
	}
	return append(head, "sh", "-c", agentCmd)
}
```

Para o caso string única (`Recipe`) usamos `sh -c` p/ tolerar comandos multi-palavra; para preservar fronteiras de argumento, `sbx shell` (Task 5.4) usa `Prefix(...)` + o argv cru do agente. **Como validar:** `ai-jail --no-docker --rw-map <sock> -- printf '%s\n' a "b c"` deve imprimir `a` e `b c` em linhas separadas (fronteiras preservadas quando passado argv cru).

// append to sbx/internal/session/session.go
// RuntimeDir returns the per-session runtime directory that holds the
// product's own podman API socket (never the host Docker socket).
func RuntimeDir(sessionID string) string {
	base := os.Getenv("XDG_RUNTIME_DIR")
	if base == "" {
		base = filepath.Join(os.TempDir(), "sbx-run")
	}
	return filepath.Join(base, "sbx", sessionID)
}

// SocketPath returns the product's podman socket for a session. This is the
// ONLY socket exposed into the ai-jail via --rw-map.
func SocketPath(sessionID string) string {
	return filepath.Join(RuntimeDir(sessionID), "api.sock")
}

// append to sbx/internal/driver/podman.go
// ServiceArgs builds the argv that runs the product's podman API socket over
// the SAME own storage roots (via baseArgs). The Docker-compat REST API served
// here is what the jailed agent's sbx talks to — never the host Docker socket.
func (p *Podman) ServiceArgs(socketPath string) []string {
	return append(p.baseArgs(), "system", "service", "--time=0", "unix://"+socketPath)
}

// StartService starts `podman system service` bound to socketPath and returns
// the running process. The caller owns its lifetime (Kill on exit).
func (p *Podman) StartService(ctx context.Context, socketPath string) (*exec.Cmd, error) {
	if err := os.MkdirAll(filepath.Dir(socketPath), 0o700); err != nil {
		return nil, DriverError{Code: "socket_dir_failed", Message: err.Error(),
			Hint: "cannot create the product socket directory under $XDG_RUNTIME_DIR"}
	}
	cmd := exec.CommandContext(ctx, p.bin, p.ServiceArgs(socketPath)...)
	if err := cmd.Start(); err != nil {
		return nil, DriverError{Code: "service_failed", Message: err.Error(),
			Hint: "podman system service failed to start; verify rootless podman with `podman info`"}
	}
	return cmd, nil
}
```

// modify the Podman struct + NewPodman + baseArgs in sbx/internal/driver/podman.go
type Podman struct {
	bin       string
	root      string
	runroot   string
	remoteURL string
}

func NewPodman(stateDir string) *Podman {
	return &Podman{
		bin:     "podman",
		root:    filepath.Join(stateDir, "storage"),
		runroot: filepath.Join(stateDir, "runroot"),
	}
}

// NewPodmanRemote talks to a product podman API socket instead of driving local
// storage. Used by the agent INSIDE the jail, where the only reachable engine is
// the product's socket mapped in via --rw-map. It never references a host socket.
func NewPodmanRemote(socketURL string) *Podman {
	return &Podman{bin: "podman", remoteURL: socketURL}
}

func (p *Podman) baseArgs() []string {
	if p.remoteURL != "" {
		return []string{"--remote", "--url", p.remoteURL}
	}
	return []string{"--root", p.root, "--runroot", p.runroot}
}

// sbx/internal/cli/shell.go
package cli

import (
	"context"
	"os"
	"os/exec"
	"time"

	"github.com/gustavocarvalho/sbx/internal/aijail"
	"github.com/gustavocarvalho/sbx/internal/session"
	"github.com/spf13/cobra"
)

// shellPlan is the deterministic, unit-testable assembly of the jail invocation:
// aijail.Prefix (up to "--") + the raw agent argv (preserving arg boundaries).
func shellPlan(socketPath string, agentArgs []string, opts ...aijail.Option) ([]string, error) {
	if err := aijail.ValidateSocket(socketPath); err != nil {
		return nil, err
	}
	head := aijail.Prefix(socketPath, opts...)
	if len(agentArgs) == 0 {
		agentArgs = []string{"bash"}
	}
	return append(head, agentArgs...), nil
}

func newShellCmd() *cobra.Command {
	var timeout time.Duration
	var mask []string
	c := &cobra.Command{
		Use:   "shell [-- <agent> [args...]]",
		Short: "Launch an agent under ai-jail wired to the product's podman engine",
		Long: `shell brings up the product's isolated podman engine for this session,
exposes ONLY its own socket into an ai-jail sandbox (--no-docker, .env* masked),
and runs the agent there. The host Docker socket and host credentials are never
exposed. On exit (or --timeout) every environment in the session is destroyed.`,
		Example: `  # interactive shell inside the jail, wired to the product engine
  sbx --session demo shell

  # run Claude Code inside the jail
  sbx --session demo shell -- claude

  # cap the session lifetime and mask an extra secrets dir
  sbx --session demo shell --timeout 45m --mask 'secrets/' -- claude`,
		RunE: func(cmd *cobra.Command, args []string) error {
			d := depsFrom(cmd)
			var opts []aijail.Option
			if len(mask) > 0 {
				opts = append(opts, aijail.WithMask(mask...))
			}
			return runShell(cmd.Context(), d, args, timeout, opts...)
		},
	}
	c.Flags().DurationVar(&timeout, "timeout", 0, "auto-destroy the session after this duration (0 = no timeout)")
	c.Flags().StringArrayVar(&mask, "mask", nil, "extra path patterns to hide from the agent (in addition to .env*)")
	return c
}

// servicer is the subset of the podman driver that can expose its socket.
type servicer interface {
	StartService(ctx context.Context, socketPath string) (*exec.Cmd, error)
}

func runShell(ctx context.Context, d deps, agentArgs []string, timeout time.Duration, opts ...aijail.Option) error {
	socketPath := session.SocketPath(d.session)

	// 1. Security gate: never expose a host docker socket.
	if err := aijail.ValidateSocket(socketPath); err != nil {
		return err
	}

	// 2. The active driver must be able to expose a socket (host-side podman).
	svc, ok := d.drv.(servicer)
	if !ok {
		return CLIError{Code: "engine_no_socket",
			Message: "the active driver cannot expose a socket",
			Hint:    "run `sbx shell` on the host with the podman driver (unset SBX_DRIVER/SBX_PODMAN_URL)"}
	}

	// 3. ai-jail must be installed.
	if _, err := exec.LookPath("ai-jail"); err != nil {
		return CLIError{Code: "aijail_missing",
			Message: "ai-jail not found on PATH",
			Hint:    "install ai-jail (github.com/akitaonrails/ai-jail) and ensure it is on PATH"}
	}

	// 4. Start the product podman service on the session socket.
	svcCmd, err := svc.StartService(ctx, socketPath)
	if err != nil {
		return err
	}
	defer func() {
		if svcCmd.Process != nil {
			_ = svcCmd.Process.Kill()
		}
		_ = os.Remove(socketPath)
	}()

	// 4b. In-process session supervisor role (M4): open registry, start egress
	//     proxy, publish ProxyAddr so `sbx env create` inside the jail injects
	//     ProxyEnv (M4.5). Without this, M3's --internal network has no egress.
	reg, err := session.OpenRegistry(d.session)
	if err != nil {
		return err
	}
	if startSessionProxy != nil {
		addr, err := startSessionProxy(d.session)
		if err != nil {
			return err
		}
		_ = reg.SetProxy(addr)
		_ = reg.SetSupervisor(os.Getpid(), time.Time{})
		if timeout > 0 {
			_ = reg.SetSupervisor(os.Getpid(), time.Now().Add(timeout))
		}
	}
	defer func() {
		_ = session.DestroyAll(context.Background(), d.drv, d.session)
		if stopSessionProxy != nil {
			_ = stopSessionProxy(d.session)
		}
		if reg2, err := session.OpenRegistry(d.session); err == nil {
			_ = reg2.MarkEnded()
		}
	}()

	// 5. Build the ai-jail argv (raw agent argv preserves boundaries).
	argv, err := shellPlan(socketPath, agentArgs, opts...)
	if err != nil {
		return err
	}

	// 6. Optional timeout: cancel the run context after the deadline.
	runCtx := ctx
	if timeout > 0 {
		var cancel context.CancelFunc
		runCtx, cancel = context.WithTimeout(ctx, timeout)
		defer cancel()
	}

	// 7. Run the agent under ai-jail. Its sbx points at the product socket via
	//    SBX_PODMAN_URL / CONTAINER_HOST.
	jail := exec.CommandContext(runCtx, argv[0], argv[1:]...)
	jail.Stdin, jail.Stdout, jail.Stderr = os.Stdin, os.Stdout, os.Stderr
	jail.Env = append(os.Environ(),
		"SBX_PODMAN_URL="+socketPath,
		"CONTAINER_HOST=unix://"+socketPath,
		"SBX_SESSION="+d.session,
	)
	return jail.Run()
	// cleanup runs via defer above (DestroyAll + stop proxy + MarkEnded)
}

- `go test ./...` verde (unit determinístico, sem engine); `go test -tags integration ./...` verde ou SKIP conforme `ai-jail`/`podman`.
- `sbx shell [-- <agente>]` sobe o serviço podman do produto num socket próprio da sessão, sobe o **proxy de egresso in-process** (papel de supervisor M4: `SetProxy` no registry), e roda o agente sob `ai-jail --no-docker --rw-map <socket> --mask .env*`, com o agente alcançando **apenas** o engine do produto; o socket do Docker do host e as credenciais do host permanecem **invisíveis** (verificado no teste de integração e no roteiro manual).
- Cleanup ao sair usa `session.DestroyAll` + stop proxy + `MarkEnded` (contrato M4) — **sem** helper local `List`+`Destroy`.

- `sbx bootstrap-claude-md` imprime um bloco CLAUDE.md colável documentando as primitivas (`create/exec/logs/status/destroy`, `shell`), as regras (`.env.sandbox`, portas dinâmicas via `sbx env status --json`, allowlist de rede) e o fluxo create→exec→logs→destroy.
